# Evidence Notes Template
> How to record findings so later turns can build on them.

Fill this skeleton for every inspection answer, most load-bearing facts first.

- **Question:** the exact thing being asked, in one line.
- **Verdict:** the direct answer in 1–2 sentences, stated with confidence level (confirmed / likely / unknown).
- **Evidence:** one bullet per fact, each anchored to its source — file, symbol or section, and roughly where (function name, heading, or line range). Quote the decisive line verbatim when it settles the question.
- **Not examined:** files or paths that were relevant but unread, so no one mistakes silence for absence.
- **Open questions:** anything the evidence could not settle, phrased as the next concrete thing to read.
- **Trail:** ordered list of everything read this turn, so a later turn can resume without re-reading.

## Rules
- Separate observation ("the code does X") from inference ("so Y probably holds") — label inferences.
- Never cite a file from memory; only from what was actually read this session.
