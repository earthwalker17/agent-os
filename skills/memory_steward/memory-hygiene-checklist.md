# Memory Hygiene Checklist
> Keeping memory files readable as they grow.

Run these checks whenever a memory file is touched or feels bloated.

1. **Dedupe** — search for the fact before adding it; update the existing entry instead of appending a near-duplicate.
2. **Supersede, don't stack** — when a decision or status changes, rewrite the entry to the new truth; keep at most one line of history ("previously X, changed because Y").
3. **Prune stale state** — delete done tasks from the queue, resolved blockers from status, and dead research threads. Archive only if the rationale still matters.
4. **One idea per bullet** — split compound entries; a reader should scan headlines, not parse paragraphs.
5. **Keep sections honest** — decisions in decisions, tasks in the queue; move misfiled entries rather than duplicating them.
6. **Cap length** — if a file exceeds roughly one screen per section, compress the oldest material into a summary line.
7. **Date what decays** — timestamp status entries and external facts that can go stale.

Caveat: never rewrite the record of *why* a decision was made — rationale is the one thing worth keeping verbatim.
