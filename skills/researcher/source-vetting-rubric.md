# Source Vetting Rubric
> Which sources to trust, and how much, for what.

Score each source 0–2 on the criteria below; prefer sources totaling 7+, drop anything under 4.

1. **Authority** — Good: primary source (official docs, spec, paper, maintainer post). Weak: anonymous blog, aggregator, AI-generated content farm.
2. **Recency** — Good: dated, current for the topic's rate of change (APIs: <1 yr; fundamentals: any). Weak: undated or stale versions.
3. **Verifiability** — Good: claims backed by data, code, or citations you can check. Weak: assertions with no trail.
4. **Independence** — Good: no incentive to oversell (vendor benchmarks count as marketing). Weak: sales pages, sponsored posts.
5. **Corroboration** — Good: key claim appears in 2+ independent sources. Weak: single-source claims — flag them as such.

Match trust to use: official docs for how-it-works, issue trackers for known bugs, benchmarks only from reproducible setups.

Caveat: high scores reduce, never remove, the need to cite — every claim keeps its source.
