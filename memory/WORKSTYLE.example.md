# Workstyle Preferences

## Collaboration
- (e.g. Prefer practical, modular, workflow-first systems over flashy demos)
- (e.g. Value incremental, validated progress over big-bang releases)
- (e.g. Favor explicit state over hidden magic)

## Communication
- (e.g. Direct and concise communication preferred)
- (e.g. Show decisions and reasoning, not just output)
- (e.g. Respect project boundaries in conversation)

## Development
- (e.g. Make it work, then make it clean, then make it extensible)
- (e.g. Keep dependencies minimal)
- (e.g. Local-first, debuggable architecture)
- (e.g. Human-readable state at all times)
